## 1. 项目概述

### 1.1 项目定位

**项目名称**：KiloCode for Obsidian（暂定）  
**项目类型**：Obsidian 社区插件（Desktop Only）  
**核心价值**：将 KiloCode AI 编码代理嵌入 Obsidian 侧边栏，让 Vault 成为 AI 的工作目录

### 1.2 目标用户

- 使用 Obsidian 进行知识管理的开发者
- 希望在笔记环境中直接与 AI 协作的用户
- KiloCode 用户

### 1.3 实施策略

**方案 C：全新项目 + 混合借鉴**

- **直接复制**：Claudian 的 UI 组件、i18n、工具函数、Chat View
- **独立重写**：Provider 层（KiloCode CLI 适配器）
- **独立项目**：新仓库、新品牌、独立发布

## 3. 功能范围

### 3.1 核心功能（对标 Claudian）

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 侧边栏聊天（多 Tab） | P0 | 基础聊天界面，支持多标签页 |
| 流式响应 | P0 | 实时显示 AI 回复 |
| 会话历史 | P0 | 保存/恢复/删除对话 |
| Inline Edit | P1 | 选中文本 + 快捷键编辑 |
| Slash Commands | P1 | `/` 触发可复用提示模板 |
| @mention | P1 | 提及 Vault 文件 |
| Plan Mode | P2 | Shift+Tab 切换计划模式 |
| MCP Server | P2 | 连接外部工具 |
| i18n | P2 | 多语言支持 |

### 3.2 功能详情

#### 侧边栏聊天（P0）

**入口**：
- 功能区图标（Bot 图标）
- 命令面板：`KiloCode: Open chat view`

**布局**：
- 支持左/右侧边栏 + 主标签页三种放置位置
- 多 Tab 管理（最多 10 个标签页）
- 每个 Tab 独立会话

**消息渲染**：
- Markdown 渲染（代码高亮、表格、列表）
- 流式打字效果
- 工具调用可视化（文件读写、搜索、命令执行）
- 图片附件支持

**交互**：
- Enter 发送，Shift+Enter 换行
- @提及文件、MCP 服务器
- / 斜杠命令
- # 指令模式

#### 流式响应（P0）

**实现方式**：
- KiloCode CLI 通过 JSON-RPC 推送流式更新
- Plugin 接收后实时渲染到 UI
- 支持中断（Cancel）操作

**消息类型**：
| 类型 | 说明 |
|------|------|
| `text` | 文本内容（流式追加） |
| `tool_use` | 工具调用（文件操作、搜索等） |
| `tool_result` | 工具执行结果 |
| `error` | 错误信息 |
| `done` | 完成信号 |

#### 会话管理（P0）

**存储位置**：`vault/.kilocode/` 目录

**功能**：
| 功能 | 说明 |
|------|------|
| 创建会话 | 新 Tab 或命令触发 |
| 恢复会话 | 从历史记录恢复 |
| 删除会话 | 清理不需要的对话 |
| 重命名会话 | 自定义标题 |
| Fork 会话 | 从某个节点分叉 |
| 压缩会话 | 压缩历史以节省 token |

#### Inline Edit（P1）

**触发方式**：
- 选中文本 + 快捷键（Ctrl/Cmd + Shift + E）
- 光标位置 + 快捷键

**交互流程**：
1. 用户选中文本或定位光标
2. 弹出编辑模态框
3. 用户输入编辑指令
4. AI 返回编辑结果
5. 词级 diff 预览
6. 用户 Accept 或 Reject

#### Slash Commands（P1）

**触发**：输入 `/` 显示命令列表

**内置命令**：
| 命令 | 说明 |
|------|------|
| `/compact` | 压缩会话历史 |
| `/clear` | 清空当前会话 |
| `/model` | 切换模型 |
| `/mode` | 切换模式（plan/code/ask） |

#### @mention（P1）

**触发**：输入 `@` 显示可提及列表

**可提及类型**：
| 类型 | 说明 |
|------|------|
| Vault 文件 | 将文件内容作为上下文 |
| MCP 服务器 | 连接外部工具 |
| 子代理 | 调用其他 AI 代理 |
| 外部目录 | 引入外部文件 |

### 5.2 数据流

#### 发送消息
```
用户输入 → InputController → ChatRuntime → KiloCode CLI
```

#### 流式响应
```
KiloCode CLI → StreamController → UI 渲染
```

#### 工具调用
```
AI 请求 → Adapter → Obsidian API → 结果返回 AI
```

### 5.3 通信协议（JSON-RPC over stdio）

**请求格式**：
```json
{
  "jsonrpc": "2.0",
  "method": "sendMessage",
  "params": {
    "content": "用户消息",
    "context": {
      "vaultPath": "/path/to/vault",
      "currentNote": "note.md"
    }
  },
  "id": 1
}
```

**响应格式（流式）**：
```json
{
  "jsonrpc": "2.0",
  "method": "stream",
  "params": {
    "type": "text",
    "content": "AI 回复内容...",
    "done": false
  }
}
```

## 7. 错误处理与用户体验

### 7.1 错误分级

| 级别 | 颜色 | 示例 | 处理方式 |
|------|------|------|----------|
| **致命** | 🔴 红色 | CLI 未安装、进程崩溃 | 全屏提示 + 重试按钮 |
| **警告** | 🟡 黄色 | 网络超时、工具失败 | 内联提示 + 自动重试 |
| **信息** | 🔵 蓝色 | 会话恢复、状态变更 | 轻量提示 |

### 7.2 关键错误场景

| 场景 | 处理方式 |
|------|----------|
| CLI 未安装 | 提示安装命令 + 重试按钮 |
| CLI 启动失败 | 显示错误 + 手动配置入口 |
| 网络错误 | 自动重试 3 次 + 取消按钮 |
| 工具调用失败 | 内联错误 + 重试/跳过 |

### 7.3 加载状态

| 状态 | UI 表现 |
|------|---------|
| 流式响应 | 光标闪烁 + Cancel 按钮 |
| 工具执行 | 黄色旋转图标 |
| 操作成功 | 绿色勾号 + 耗时 |

### 7.4 空状态

| 场景 | 设计 |
|------|------|
| 无会话 | 欢迎界面 + 快捷键提示 |
| 会话列表空 | 空状态插画 + 创建提示 |

### 7.5 快捷键

| 操作 | 快捷键 |
|------|--------|
| 发送消息 | Enter |
| 换行 | Shift+Enter |
| Inline Edit | Ctrl/Cmd+Shift+E |
| Plan Mode | Shift+Tab |
| 取消 | Esc |
| 新标签页 | Ctrl/Cmd+T |

### 7.6 性能优化

| 优化点 | 策略 |
|--------|------|
| 消息渲染 | 虚拟滚动 |
| 文件加载 | 延迟加载 |
| 会话恢复 | 按需加载 |
| 工具调用 | 异步执行 |
